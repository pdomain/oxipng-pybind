# Next Work Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move from proved CI/release automation to release-ready dependency,
PyPI, and notice automation.

**Architecture:** Keep the stable API unchanged. Add small automation scripts
and workflow gates that classify dependency impact, verify release artifacts,
and reduce hand-maintained compliance data. Each task must land as a focused PR
with GitHub Actions checks passing before merge.

**Tech Stack:** GitHub Actions, `uv`, Cargo, `cargo metadata`/`cargo tree`,
`tomllib`, `pytest`, `ruff`, `basedpyright`, `maturin`, PyPI Trusted
Publishing.

---

## Task 1: Dependency Refresh Release Classification

**Files:**

- Create: `scripts/classify_dependency_refresh.py`
- Modify: `.github/workflows/dependency-health.yml`
- Modify: `docs/process/dependency-health.md`
- Test: `tests/test_dependency_refresh_classification.py`
- Test: `tests/test_workflows.py`

- [ ] **Step 1: Add classifier tests**

Create tests that cover:

- `uv.lock`-only tooling changes produce `no-release-needed`.
- `Cargo.lock` changes reachable from the shipped Rust graph produce
  `release-needed`.
- `[project.dependencies]` changes produce `release-needed`.
- The workflow uses classifier outputs for PR labels and auto-merge gating.

Run:

```bash
uv run --no-sync --group dev pytest tests/test_dependency_refresh_classification.py tests/test_workflows.py -q
```

Expected before implementation: classifier tests fail because the script does
not exist.

- [ ] **Step 2: Implement the classifier**

Build a stdlib-first script that compares the refreshed workspace against the
base ref and emits GitHub outputs:

- `release-needed=true|false`
- `label=release-needed|no-release-needed`
- `reason=<human-readable reason>`

Use Cargo graph inspection for Rust runtime/build dependency reachability and
`tomllib` for Python runtime dependencies.

- [ ] **Step 3: Wire dependency-health**

Run the classifier in the prepare job after lockfile refresh and before
artifact upload. Include the label and reason in the dependency refresh PR body.
Enable auto-merge only for `no-release-needed` PRs. Leave `release-needed` PRs
open for an explicit wrapper post-release bump before merge.

- [ ] **Step 4: Verify**

Run:

```bash
uv run --no-sync --group dev ruff check scripts/classify_dependency_refresh.py tests/test_dependency_refresh_classification.py tests/test_workflows.py
uv run --no-sync --group dev basedpyright scripts/classify_dependency_refresh.py tests/test_dependency_refresh_classification.py tests/test_workflows.py
uv run --no-sync --group dev pytest tests/test_dependency_refresh_classification.py tests/test_workflows.py -q
```

- [ ] **Step 5: Hosted validation**

Open a PR, let required checks pass, merge, then run:

```bash
gh workflow run dependency-health.yml --repo pdomain/oxipng-pybind --ref main
```

Confirm a tooling-only refresh gets `no-release-needed` and auto-merges after
required checks pass.

- [ ] **Step 1: Decide artifact policy**

documented and tested.

- [ ] **Step 2: Add Trusted Publishing workflow**

- [ ] **Step 3: Verify workflow policy**

Add static workflow tests for:

- tag-only publish trigger;
- no password/API-token publishing secret;
- artifact verification before publish;
- no publish on arbitrary PRs.
